# Internationalization API

Complete API reference for internationalization endpoints.

## Base Path
`/api/i18n`

## Overview

The i18n API provides access to translation files for different languages. Translation files are stored as JSON in the `locales/` directory.

## Endpoints

### GET `/api/i18n/list`
Get list of available languages.

**Response:** `200 OK`
```json
["en", "de"]
```

**Error Responses:**
- `500`: Server error

**Notes:**
- Returns language codes (file names without `.json` extension)
- Results are sorted alphabetically
- Only `.json` files in the locales directory are included

---

### GET `/api/i18n/:language`
Get translations for a specific language.

**Path Parameters:**
- `language` (string, required): Language code (e.g., `en`, `de`, `fr`)

**Response:** `200 OK`
```json
{
  "app": {
    "title": "SpoolDB"
  },
  "filament": {
    "add": "Add Filament",
    "name": "Name",
    "material": "Material",
    "color": "Color"
  },
  "common": {
    "save": "Save",
    "cancel": "Cancel",
    "delete": "Delete"
  }
}
```

**Error Responses:**
- `400`: Invalid language code (only alphanumeric and hyphens allowed)
- `404`: Language not found
- `500`: Server error

**Security Notes:**
- Language code is validated to prevent directory traversal attacks
- Only alphanumeric characters and hyphens are allowed
- File path is resolved and validated to ensure it's within the locales directory

## Supported Languages

Currently supported:
- **English** (`en`)
- **German** (`de`)

## Adding New Languages

1. Create a new JSON file in the `locales/` directory (e.g., `fr.json`)
2. Follow the same structure as existing translation files
3. The language will automatically appear in the list endpoint

## Rate Limiting

Both endpoints use the strict rate limiter: **200 requests per 15 minutes per IP**.
